웹프로그래밍-I 교과목 학기말 과제
1. 과제 내용- 2인 1팀을 구성하여 다음 과제 주제 중에 하나를 선택하여 구현하도록 한다.- 과제 주제별로 제시된 핵심 기능 외의 기능에 대해서는 각 팀에서 추가 제안하여 
구현하도록 한다.
(1) 스터디룸 및 사물함 예약 시스템- 설명: 사용자가 날짜와 시간을 선택하여 원하는 스터디룸이나 사물함을 예약하고 
결제하는 시스템- 핵심 기능:
• 회원 관리: 일반 회원 및 관리자(Admin) 권한 분리 구현.
• 예약 로직: 중복 예약 방지를 위한 데이터베이스 트랜잭션 처리 
(Synchronized 또는 DB Lock 활용).
• 마이페이지: 본인의 예약 현황 확인, 취소 요청 기능.
• 관리자 페이지: 좌석 배치 현황 조회, 매출 통계 기능.
(2) 대학생 중고 전공서적 및 필기 공유 거래 장터- 설명: 대학생들이 중고 전공책을 거래하거나, 자신이 작성한 강의 필기 노트를 업
로드하여 공유하는 플랫폼- 핵심 기능:
• 게시판(CRUD): 카테고리별(단과대/학과) 도서 등록, 이미지 파일 업로드/다운
로드 직접 구현.
• 댓글/답글: 책 상태 문의를 위한 계층형 댓글(Reply) 기능.
• 검색/필터: 도서명, 저자, 학과별 다중 조건 검색 쿼리 작성.
2. 과제 수행 방법
(1) Spring Framework 등을 사용하지 않고 pure Java(Servlet, JSP)와 템플
릿 엔진을 활용하여 웹 애플리케이션의 동작 원리(MVC 패턴)를 직접 코드
로 구현하는 것에 초점을 맞추도록 하며, 비즈니스 로직과 DB 설계에 집중
하여 구현하도록 한다.
(2) 아키텍처 및 기술 가이드라인
① Architecture: MVC Pattern (Model-View-Controller) 직접 구현
• Controller: 단일 FrontController 서블릿을 두어 모든 요청(*.do)을 받고, 
Command 패턴을 이용해 각 요청을 자바 클래스로 분기 처리.
• View 템플릿 엔진: JSP/EL/JSTL 조합 (가장 안정적) 또는 
Thymeleaf(Java standalone 환경용 템플릿 엔진 설정 실습 가능).
• Model: Layered architecture(Service, DAO(Data Access Object) 
그리고 DTO(Data Transfer Object)) 패턴 적용.
② Database: Connection Pool(DBCP)을 Context.xml에 직접 설정하고 Data 
Source를 통해 DB 연결을 관리
③ WEB-INF 보안: JSP 파일들은 WebContent/WEB-INF/views/ 하위에 두어 사
용자가 URL 입력을 통해 직접 접근하는 것을 방지하고, 오직 
FrontController를 통한 포워딩으로만 접근 가능하게 설계하여 구현
(3) 2인 팀이 역할을 나눌 수 있는 기능 정의서 및 일정표 제시- 기능 정의서 그리고 4주 프로젝트 일정표 및 R&R(Role and Responsibilities)
을 작성하여 수행하도록 한다.
